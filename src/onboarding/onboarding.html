What to do if the service is missing

  1. Start the local WhisperLiveKit service on port 8000.
  2. Confirm nothing else is already using localhost:8000.
  3. Click Check local readiness again after the service starts.
  4. If the check still fails, restart the local service and retry.

When the service is healthy, the extension can reach the matching WebSocket endpoint at the same host and port.

What to do if the browser is unsupported

  1. Use a Chromium-based browser with extension API support.
  2. Make sure the page is opened from the extension, not a regular tab.
  3. Reload the extension after changing browsers or updating Chrome.
  4. Then run the readiness check again.

If browser APIs are unavailable, the extension cannot request capture permissions or coordinate local session state.
