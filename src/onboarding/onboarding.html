How detection works

Onboarding checks the local HTTP endpoint first. If the endpoint responds, the extension can connect to the matching WebSocket service at the same host and port.

When the service is unavailable, the UI should stay on the local setup path and avoid starting captions until the check passes.

Recovery steps

  1. Start the local WhisperLiveKit service on port 8000.
  2. Confirm nothing else is already using localhost:8000.
  3. Reload the extension after the local service is healthy.
  4. Run this check again before starting the first caption session.
