Keep the flow on-device. When the service is reachable, the app can open captions without any cloud dependency.

Discovery only probes localhost:8000/asr. No cloud endpoints are contacted.

  • Start WhisperLiveKit on port 8000.
  • Confirm nothing else is using localhost:8000.
  • Run discovery again until the service is reachable.

What the check does

The installer probes the local HTTP endpoint first. A successful response means the matching local WebSocket service should be available at the same host and port.

Missing service

If the service is missing, start the local ASR stack and retry the discovery check from this page.

  • Use the machine running the extension.
  • Keep WhisperLiveKit on localhost:8000.
  • Retry after restarting the local service.
Local-only flow

This installer does not configure cloud services. It only discovers the local ASR endpoint used by the extension.
